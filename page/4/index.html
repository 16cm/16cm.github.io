单片机程序中为什么要禁止看门狗?

2017-03-16

设置看门狗定时器的时候，必须先关闭它才能设置，否则在开启状态下设置看门狗定时器可能会出现问题。一般单片机和DSP、还有ARM芯片里设置内部功能的时候，都是先关闭这个功能，再设置。中断也是类似的，我们通常它把中断全部关闭了，然后在程序里面进一步设置。

要搞清楚关闭看门狗定时器的深层次原因，我们首先得明白其用途：

它其实就是一个定时器，这个定时器溢出是会产生一个中断响应，这个中断响应对应的中断服务就是跳回程序开头，也就是复位。启动看门狗后，要定时在程序中把看门狗的计数器清0，以防止它进入中断然后复位，这就叫做喂狗。

看门狗定时器常用在程序很容易跑飞的情况下，比如等待超时，意外进入死循环等。在启动看门狗定时器的前提下，如果程序跑飞，不用怕，等计数器WTCNT溢出时，程序会自动复位。但在一般的程序中是不太可能会跑飞的，所以多数情况下我们关闭看门狗，是因为引入了看门狗之后就多了定时计数值处理的一些代码，这增加了程序的复杂性。

Markdown语法

2017-01-10

常用的 Markdown 语法。

01. 段落

一行就是一个段落，如果要另起一行，在两个段落之间添加一个空行即可。

02. 斜体

*斜体* -> 斜体

03. 粗体

**粗体** -> 粗体
***斜体+粗体*** -> 斜体+粗体

04. 上标

n<sup>2</sup> -> n2

05. 下标

a=log<sub>2</sub>b -> a=log2b

06. 函数

&fnof;(x)=x+1 -> ƒ(x)=x+1

07. 根号

&radic;5 -> √5

08. 角度

30&deg; -> 30°

09. 标题

# 这是 H1 #
## 这是 H2 ##
### 这是 H3 ###
...
###### 这是 H5 ######

这是 H1

这是 H2

这是 H3

…

这是 H6

Hexo指令

2017-01-09

一些常用的 Hexo 指令。

01. 新建网站

hexo init

02. 生成静态文件

hexo generate

03. 启动服务器

hexo server
默认情况下，访问网址为: http://localhost:4000/

04. 新建一篇文章

hexo new "My New Post"

05. 清除缓存文件 db.json 和已生成的静态文件 public

hexo clean

06. 部署网站

hexo deploy

07. 显示草稿

hexo --draft
显示 source/_drafts 文件夹中的草稿文章。

08. 显示版本

hexo -v

09. 安全模式

hexo --safe
在安全模式下，不会载入插件和脚本。当您在安装新插件遭遇问题时，可以尝试以安全模式重新执行。

10. 调试模式

hexo --debug
在终端中显示调试信息并记录到 debug.log。

Hexo博客搭建教程

2017-01-08

基于 Github 搭建 Hexo 框架的博客，快速上手而且简洁高效，详见 Hexo文档。

1. Node.js

下载链接：32bit&64bit
不确定版本的话直接下载32位即可，然后保持默认设置安装。
检查安装情况，win+r 打开运行，输入 cmd 回车，输入以下代码:
node -v
npm -v
如下图所示，则说明安装正确。

2. Git

下载链接：32bit&64bit
其他设置只需要保持默认，但建议 PATH 选项按照下图选择：

检查安装情况，输入以下代码:
git --version
如下图所示，则说明安装正确。

3. Github

打开 https://github.com/ 注册帐号，前往邮箱确认注册。
登录帐号，点击页面右上角的+，选择 New repository，注意在 Repository name 下必须填写为 yourname.github.io，其他可不填，然后点击 Create repository。

4. GitHub Pages

可以去看这里的官方教程，只要 yourname.github.io 可以正常访问就可以。

5. Hexo

任意盘符下新建文件夹，打开文件夹右键选择 Git Bash Here，如图：

输入：
npm install hexo-cli -g
看到下图，继续输入：

npm install hexo --save
结束后输入：
hexo -v
看到下图就说明安装成功。

输入以下代码初始化：
hexo init
然后输入：
npm install
等待 npm 安装完所需组件即可。
安装完成后，再次输入：
hexo g
然后
hexo s
这时在浏览器中打开 http://localhost:4000/，就看见了你的博客首页。

6. 部署博客

打开命令行窗口，然后输入：
git config --global user.name "yourname"
git config --global user.email "youremail"
然后在_config.yml 文件中，找到 Deployment，然后按照如下格式修改：
deploy:
 type: git
 repo: git@github.com:yourname/yourname.github.io.git
 branch: master
再输入：
ssh-keygen -t rsa -C "youremail@gmail.com"
连续按3个回车后返回如下信息：
Your identification has been saved in /home/tekkub/.ssh/id_rsa.
Your public key has been saved in /home/tekkub/.ssh/id_rsa.pub.
打开生成的 id_rsa.pub，前往 https://github.com/settings/keys 添加密钥。
输入以下代码测试：
ssh git@github.com
返回 successfully 信息即表示部署成功。
我们就可以使用以下命令发布文章了：
hexo d
