搭建Telegram的MTProxy代理

Dec 08 2018

1. 安装依赖开发包openssl和zlib

1.1. 在CentOS上:
yum install openssl-devel zlib-devel
yum groupinstall "Development Tools"
1.2. 在Debian / Ubuntu上：
apt install git curl build-essential libssl-dev zlib1g-dev

2. 克隆MTProxy的源码

git clone https://github.com/TelegramMessenger/MTProxy
cd MTProxy

3. 编译安装，安装的位置在objs/bin/mtproto-proxy

make && cd objs/bin

4. 运行程序

4.1 获取一个密钥，用于连接到电报服务器
curl -s https://core.telegram.org/getProxySecret -o proxy-secret
4.2 获取当前的Telegram的配置
curl -s https://core.telegram.org/getProxyConfig -o proxy-multi.conf
4.3 生成一个密钥供其他用户连接到您的代理使用
head -c 16 /dev/urandom | xxd -ps
4.4 运行mtproto-proxy服务器
服务器有内网IP，如谷歌云、微软云、阿里云的需要设置NAT，请用第二条启动命令。
无内网IP的启动命令，secret 这里替换为之前生成的密钥。
./mtproto-proxy -u nobody -p 8888 -H 443 -S secret --aes-pwd proxy-secret proxy-multi.conf -M 1
有内网IP的启动命令，secret 这里替换为之前生成的密钥，内网IP:公网IP 替换为你服务器的内网IP和公网IP。
./mtproto-proxy -u nobody -p 8888 -H 443 -S secret --aes-pwd proxy-secret proxy-multi.conf -M 1 --nat-info 内网IP:公网IP
443为端口号，可以自定义， secret 替换为第三步生成的密钥。

5. 生成Telegram的代理快速链接

tg://proxy?server=SERVER_NAME&port=PORT&secret=SECRET
SERVER_NAME修改为你服务器的地址，PORT修改为你的端口号，SECRET修改为之前生成的密钥。

6. 服务器重启后如何重新启动代理服务

6.1 进入代理服务器启动目录
cd MTProxy/objs/bin
6.2 服务器有内网IP，如谷歌云、微软云、阿里云的需要设置NAT，请用第二条启动命令。
无内网IP的启动命令，secret 这里替换为之前生成的密钥。
./mtproto-proxy -u nobody -p 8888 -H 443 -S secret --aes-pwd proxy-secret proxy-multi.conf -M 1
有内网IP的启动命令，secret 这里替换为之前生成的密钥，内网IP:公网IP替换为你服务器的内网IP和公网IP。
./mtproto-proxy -u nobody -p 8888 -H 443 -S secret --aes-pwd proxy-secret proxy-multi.conf -M 1 --nat-info 内网IP:公网IP
